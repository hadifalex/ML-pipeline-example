The final part of the course comprises a project that applies code and concepts across all parts of the L2D course, to a given dataset.

### üìà Data Options:

You have three main options for approaching this project. Either:

1. You can use our sample datasets, included in this repository, in the [data](/data/) folder.
2. You can use your own data, if you deem it suitable to demonstrate the analyses required by the [Final Project](Final_Project_Template.ipynb).
3. You can use a public dataset of your choosing, that you deem suitable.

**Note**: If you opt for options 2 or 3, and are unsure regarding the suitability of your data, please [contact us](mailto:admin@learntodiscover.ai), and we will do our best to advise you. If your data are too large (more than 100mb) to upload to this GitHub repository, you will be responsible for hosting them online in a way that we can access them.

### üèÅ Completing your Final Project:

Please take a moment to thoroughly look through the contents of this GitHub repository. We have provided you with a [**Final Project Template**](Final_Project_Template.ipynb), that you should fill out and submit, as you would an ordinary assignment, by committing and synchronising your changes back to your GitHub repository.

The template contains instructions, guidelines, and various checkpoints that you should aim to satisfy, in order to complete the Project to the fullest extent.

### üìÅ Example, Model Project:

For one of our provided sample datasets (EEG with annotated eye state) you can refer to our example [**model project**](Resources/model_project_eeg.ipynb), to understand what is required and expected.

### üí° Tips:

There is no 'right answer' for a successful Final Project. We are essentially just looking to assess that you have understood the breadth of topics covered on the course, with the Project intending to help you think about how to transfer these methods and techniques, in combination, to a real-world example. Please feel free to approach this with your own crtical thinking, providing written explanation in your Project notebook, in the form of well-commented code, and text explanation in markdown cells, if and when appropriate. 

### üì© Project Submission and Deadline:

As aforementioned, please approach submission as you would with a regular L2D assignment - namely by working on the [**Final Project Template**](Final_Project_Template.ipynb) notebook (either in a Codespace or IDE of your choice), and committing changes, and synchronising or pushing these back to this repository. Please add a sensible commit message (such as: "Final Project submitted." or simialr), when submitting your notebook. 

The deadline for this project is: **January 31st, 2026**. If you are struggling to meet this deadline, for any reason, please contact us, and let us know, so we can make arrangements.
